# 视频照片格式

拍摄前先想清楚「以什么格式记录」，直接决定了后期能有多大的调整空间、文件有多大、以及素材好不好用。本文分照片与视频两部分，介绍最常用的格式，并给出**部门活动的统一拍摄规范**。

---

## 照片格式

相机存储照片主要有两种思路：一种是相机替你处理好、直接能用的 **JPEG**；另一种是保留传感器原始数据、交给后期处理的 **RAW**。

### JPEG

JPEG 是相机内部已经完成处理（白平衡、对比度、锐化、色彩）并压缩好的成品图。

- **优点**：文件小、兼容性极好，任何设备都能直接打开和分享，拍完即用
- **缺点**：8 bit 色深、有损压缩，相机的处理已经「烤」进画面，后期调整空间很小——大幅提亮暗部或拉回过曝高光时容易出现断层和噪点

适合对画质宽容度要求不高、需要快速交付的场合（如现场快速出图）。

### RAW

RAW 记录传感器捕捉到的**原始数据**，几乎不做处理。它不是一张「能直接看」的成品图，而是后期的「数字底片」。

- **优点**：12–14 bit 高色深，保留最大动态范围；白平衡、曝光、色彩都可在后期无损调整；能从看似过曝/欠曝的画面中找回大量细节
- **缺点**：文件大（通常是 JPEG 的数倍）；必须经过后期软件（Lightroom、Capture One 等）处理导出；各家格式不通用

> RAW 没有统一后缀：佳能为 **.CR3 / .CR2**，尼康为 **.NEF**，索尼为 **.ARW**，富士为 **.RAF**，通用格式为 **.DNG**。

### JPEG vs RAW

| 对比项 | JPEG | RAW |
|--------|------|-----|
| 色深 | 8 bit | 12–14 bit |
| 压缩 | 有损 | 无损 / 轻度压缩 |
| 后期宽容度 | 小 | 大 |
| 文件大小 | 小 | 大 |
| 是否需后期 | 否，拍完即用 | 是，需导出 |
| 适用 | 快速分享 | 精修、留存 |

::: tip 部门拍摄规范 · 照片
**部门活动一律拍摄 RAW。**
活动照片大多需要后期调色、裁切、找回明暗细节后再使用，RAW 能最大程度保留素材的可调整空间，避免现场曝光 / 白平衡失误无法挽回。若机身支持 **RAW + JPEG 同时记录**，可一并开启，方便现场预览与快速选片。
:::

## 视频格式

视频格式要先分清两个经常被混为一谈的概念：

- **编码（Codec）**：视频数据如何被压缩，决定画质与文件大小，如 **H.264、H.265**
- **封装（Container）**：把视频、音频、字幕、元数据装进一个文件的「盒子」，对应文件后缀，如 **.mov、.mp4**

一个 `.mp4` 文件里装的可能是 H.264，也可能是 H.265——后缀只代表盒子，不代表编码。

### 编码：H.264 与 H.265

**H.264（AVC）**
目前兼容性最好的编码，几乎所有设备、平台、剪辑软件都能直接读取和硬件解码。压缩效率成熟，多为 8 bit，是分享与交付的安全选择。

**H.265（HEVC）**
新一代编码，**相同画质下文件约比 H.264 小一半**，支持 10 bit 与更高动态范围。代价是编解码更吃性能，老旧设备或软件兼容性稍差。

| 对比项 | H.264 | H.265 |
|--------|-------|-------|
| 压缩效率 | 一般 | 高（约省 50% 体积） |
| 色深 | 通常 8 bit | 支持 10 bit |
| 兼容性 | 极好 | 较好，老设备可能不支持 |
| 解码 / 剪辑负担 | 低 | 高 |

### 封装：MOV 与 MP4

- **MOV**：苹果 QuickTime 封装，专业拍摄与剪辑流程常用，常用于承载 Log、ProRes 等高质量素材
- **MP4**：通用性最强的封装，适合网络上传、播放与分享，多搭配 H.264 / H.265

两者都能装 H.264 / H.265，日常差别不大；**专业拍摄优先 MOV，分享交付优先 MP4**。

### Log 模式

**Log** 是一种「拉平」的画面记录方式：直出画面看起来灰、低对比、低饱和，但保留了最大的动态范围（高光与暗部细节）。

- Log 素材**不能直接使用**，必须在后期套用 **LUT** 或手动调色还原
- 换来的是极大的调色宽容度，画面明暗与色彩可大幅调整而不崩坏
- 各家叫法不同：索尼 **S-Log**、佳能 **C-Log**、松下 **V-Log**、富士 **F-Log**

> Log 类似视频版的 RAW 思路——牺牲「拍完即用」，换取后期空间。

### 分辨率与帧率

- **1080p（Full HD）**：1920×1080，画质足够清晰，文件与剪辑压力远小于 4K，适合活动记录与社交媒体发布
- **60fps**：画面更顺滑，且为后期做慢动作（升格）留有余地

::: tip 部门拍摄规范 · 视频
**统一使用 1080p、60fps 拍摄。**

编码与封装按机器能力二选一：

1. **机器支持 Log** → 拍 **Log**，后期套 LUT 调色（动态范围与调色空间最大）
2. **机器不支持 Log** → 用 **MOV 封装**录制，编码选 **H.265**（体积小、画质好）或 **H.264**（兼容性最好）

这样既保证了活动素材的画质与后期空间，又在没有 Log 的设备上保留了稳定可用的高质量方案。
:::

## 快速对照

| 场景 | 推荐设置 |
|------|----------|
| 活动照片 | RAW（可 RAW + JPEG 双格式） |
| 视频 · 支持 Log | 1080p 60fps，Log，后期套 LUT |
| 视频 · 不支持 Log | 1080p 60fps，MOV 封装，H.265 / H.264 |
